# 岗位管理功能 v3（优化轮）- 不满意原因记录

## 产物不满意原因

### 1. 更新接口缺少参数校验
- **文件**：`backend/src/main/java/com/example/usermanagement/controller/PostController.java`
- **方法**：`updatePost(@PathVariable Long id, @RequestBody Post post)`
- **问题**：`createPost()` 有 `@Valid` 注解，但 `updatePost()` 没有。直接调用 `PUT /api/posts/{id}` 可以提交空编码或超长名称，更新时不触发校验，与新增行为不一致

### 2. SQL 权限 parent_id 错误
- **文件**：`backend/src/main/resources/sql/init.sql`
- **问题**：岗位管理权限的 parent_id 设置错误
  - "岗位管理"（post:manage）id=28，parent_id=0
  - "岗位列表"（post:list）parent_id=24，指向了"操作日志"菜单
  - "新增岗位"（post:add）parent_id=24，同样指向"操作日志"
- **影响**：岗位管理菜单在权限树中挂到了错误的父节点下，菜单显示混乱

### 3. 异常处理静默吞掉错误
- **文件**：`frontend/src/views/system/post/index.vue`
- **方法**：`loadData()`、`handleSubmit()`、`handleExport()`、`handleDelete()`、`handleBatchDelete()`
- **问题**：catch 块都是空的（`catch (e) {}`）。接口返回 400/500 错误时，前端不显示任何提示，用户不知道操作失败原因

### 4. 导出接口未携带搜索条件中的分页参数
- **文件**：`frontend/src/views/system/post/index.vue`
- **方法**：`handleExport()`
- **问题**：导出时只传了 `{ ...searchForm }`，没有传 `pageNum` 和 `pageSize`。而后端 `PostServiceImpl.java` 的 `exportPosts()` 方法调用的是 `postMapper.selectList(queryDTO)`，该查询依赖 `queryDTO.calcOffset()` 计算偏移量。由于前端没传分页参数，`pageNum` 和 `pageSize` 为 null，`calcOffset()` 可能抛出 NullPointerException，导致导出时报 500 错误

## 过程不满意原因

### 1. SQL 权限 parent_id 未修复
- **触发节点**：修改代码环节
- **实际行为**：本轮优化只修改了导出和状态管理相关代码，没有修复上一轮就存在的 SQL 权限 parent_id 错误
- **业务影响**：岗位管理菜单在权限树中仍然显示在错误位置

### 2. 更新接口 @Valid 缺失未修复
- **触发节点**：修改代码环节
- **实际行为**：本轮优化没有修复 `PostController.java` 的 `updatePost()` 缺少 `@Valid` 注解的问题
- **业务影响**：更新操作仍然可以提交非法数据

### 3. 异常处理问题未修复
- **触发节点**：修改代码环节
- **实际行为**：本轮重写了 `index.vue`，但空 catch 块问题依然存在，没有添加 `ElMessage.error()` 提示
- **业务影响**：接口报错时用户仍然看不到任何提示

### 4. 未编译验证
- **触发节点**：最终验证环节
- **实际行为**：本轮没有执行编译验证，如果 `exportPosts()` 方法中 `queryDTO.calcOffset()` 在分页参数为 null 时报错，不会被及时发现
- **业务影响**：导出功能可能在运行时报 500 错误
