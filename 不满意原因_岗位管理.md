# 岗位管理功能 - 不满意原因记录

## 产物不满意原因

### 1. 导出格式错误
- **提示词要求**：导出当前列表中的岗位数据为 Excel 文件
- **实际产物**：`frontend/src/views/system/post/index.vue` 的 `handleExport()` 方法生成的是 CSV 格式（`type: 'text/csv'`）
- **问题**：用户下载后需要用 Excel 打开 CSV，与需求要求的 Excel 格式不符

### 2. 更新接口缺少参数校验
- **文件**：`backend/src/main/java/com/example/usermanagement/controller/PostController.java`
- **方法**：`updatePost(@PathVariable Long id, @RequestBody Post post)`
- **问题**：`createPost()` 有 `@Valid` 注解，但 `updatePost()` 没有。直接调用 `PUT /api/posts/{id}` 可以提交空编码或超长名称，更新时不触发校验，与新增行为不一致

### 3. SQL 权限 parent_id 错误
- **文件**：`backend/src/main/resources/sql/init.sql`
- **问题**：岗位管理权限的 parent_id 设置错误
  - "岗位管理"（post:manage）id=28，parent_id=0
  - "岗位列表"（post:list）parent_id=24，指向了"操作日志"菜单
  - "新增岗位"（post:add）parent_id=24，同样指向"操作日志"
- **影响**：岗位管理菜单在权限树中挂到了错误的父节点下，菜单显示混乱

### 4. 异常处理静默吞掉错误
- **文件**：`frontend/src/views/system/post/index.vue`
- **方法**：`loadData()`、`handleSubmit()`、`handleDelete()`、`handleBatchDelete()`
- **问题**：catch 块都是空的（`catch (e) {}`）。接口返回 400/500 错误时，前端不显示任何提示，用户不知道操作失败原因

## 过程不满意原因

### 1. SQL 权限 ID 计算错误
- **触发节点**：修改 `init.sql` 文件时
- **实际行为**：模型先认为权限有 27 条（ID 1-27），后来又说"当前权限总计 33 条（ID 1-33）"，实际岗位管理权限 parent_id 却指向了 24（操作日志）
- **业务影响**：岗位管理菜单在权限树中显示位置错误

### 2. 未验证导出格式
- **触发节点**：最终验证环节
- **实际行为**：模型执行了编译和启动服务，但没有实际点击导出按钮验证下载的文件格式
- **业务影响**：如果验证过，就能发现导出的是 CSV 而非 Excel

### 3. 未检查异常处理
- **触发节点**：最终验证环节
- **实际行为**：模型只通过 `view_folder` 查看了文件目录，没有检查前端代码中的异常处理逻辑
- **业务影响**：`catch (e) {}` 这种空 catch 块未被发现和修复
